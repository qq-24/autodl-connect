# AutoDL 一键连接

一个集成“AutoDL 一键开机 + 自动登录 + SSH连接”与本地端口转发的桌面应用。面向需要快速拉起云端训练环境并立刻连接开发的使用场景。

## 亮点能力

- 一键启动：在设备列表中选择目标行，自动完成开机、状态确认、页面跳转。
- 自动登录：保存账号与偏好后，启动即登录 AutoDL 控制台，可选静默/可视模式。
- SSH连接：为每台设备保存连接信息，开机并连接或直接连接均按保存项执行。
- GPU余量展示：设备表格显示 GPU 是否充足，辅助快速筛选可用资源。
- 便携存储：配置、凭据与设备映射优先保存在程序同目录，权限不足自动回退用户目录。
- 一键打包：支持打包单文件 exe，内置图标与首次运行自动释放帮助文档。

## 快速开始

1. 启动应用后，在左侧 AutoDL 面板登录（或勾选自动登录保持状态）。
2. 点击“刷新”，在设备列表中查看备注、状态、规格与 GPU余量。
3. 为目标设备点击“设置连接信息”，填写 SSH 地址、端口、用户名与密码并保存。
4. 选择“开机”或“开机并连接”，自动完成设备启动与 SSH 连接；也可直接点击“连接”。
5. 如需端口转发，在右侧备用面板粘贴 SSH 命令、设置远程端口，点击“连接”自动打开本地 URL。

## 功能详解

- 设备精准操作：限域下拉菜单与几何匹配，避免跨行误点击；支持按 `device_id / 备注` 双定位。
- 模式偏好即时保存：静默/可视浏览器模式随选随存，启动按最近选择执行。
- 连接信息弹窗：每行设备可独立编辑并保存 SSH 参数，连接动作读取对应项；缺失即提示。
- 删除全部：提供“一键清空”连接信息库的按钮，便于重置。
- 端口转发：基于 `paramiko` 建立隧道，成功后自动打开 `http://localhost:[端口]`。
- 日志与状态：关键步骤在日志区与状态区实时反馈，便于排查与复现。

## 文件与存储

- 程序目录下：
  - `configs/` 保存 SSH 连接配置与设备映射。
  - `autodl_configs/` 保存 AutoDL `cookies.json` 与 `credentials.json`。
  - `README.md` 首次运行自动释放，界面右上“？”可随时打开。
- 权限不足时，自动回退到用户目录并提示。

## 打包与首次运行

- 支持使用 PyInstaller 打包单文件 exe，并设置图标。
- 首次运行自动将内置 `README.md` 释放到程序目录，以便离线查看。
- 打包前运行 `python assets/make_icon.py` 生成新版图标。
- 打包时包含数据文件与图标：`--add-data "README.md;." --icon assets/icon.ico`。
- 可指定产物名：`--name "AutoDL 一键连接"`。

## 依赖

- Python 3.8+
- PyQt5
- paramiko
- selenium、webdriver-manager（使用 AutoDL 自动登录与设备解析时）

## 常见问题

- 登录失败：检查账号与密码；网络可达；必要时手动处理验证码。
- 开机失败：确认定位的是同一设备行；重试刷新列表；检查权限与配额。
- 连接失败：核对 SSH 参数是否完整；本地端口占用；远端服务是否可用。
