# AutoDL自动开机工具使用说明

## 功能介绍
这个工具可以自动登录AutoDL控制台，并自动点击指定设备的开机按钮，解决手动开机的繁琐操作。

## 主要功能
1. **自动登录** - 保存登录状态，避免重复输入密码
2. **设备管理** - 显示所有设备状态，支持批量选择
3. **自动开机** - 自动点击开机按钮和确认对话框
4. **状态监控** - 实时显示操作日志和状态

## 安装依赖
```bash
pip install -r requirements_autodl.txt
```

## 使用方法

### 1. 启动工具
```bash
python autodl_auto_start.py
```

### 2. 登录AutoDL
- 在"登录配置"标签页输入用户名和密码
- 点击"登录"按钮
- 工具会自动保存登录状态

### 3. 管理设备
- 切换到"设备管理"标签页
- 点击"刷新设备列表"查看所有设备
- 选择要开机的设备（可多选）
- 点击"开机选中设备"或单个设备的"开机"按钮

### 4. 自动操作
工具会自动：
- 点击开机按钮
- 等待确认对话框
- 点击确定按钮
- 刷新设备状态

## 注意事项

1. **Chrome浏览器** - 需要安装Chrome浏览器和ChromeDriver
2. **网络连接** - 确保网络连接稳定
3. **登录状态** - 登录状态会保存，下次使用无需重新登录
4. **设备选择** - 支持批量选择设备同时开机

## 配置文件

工具会在用户目录下创建配置文件夹：
- Windows: `C:\Users\用户名\.autodl_auto_starter\configs\`
- macOS: `/Users/用户名/.autodl_auto_starter/configs/`
- Linux: `/home/用户名/.autodl_auto_starter/configs/`

包含文件：
- `cookies.json` - 保存登录状态
- `devices.json` - 保存设备配置

## 故障排除

### ChromeDriver问题
如果提示找不到ChromeDriver，可以：
1. 安装webdriver-manager: `pip install webdriver-manager`
2. 手动下载对应版本的ChromeDriver并添加到PATH

### 登录失败
1. 检查用户名和密码是否正确
2. 检查网络连接
3. 查看是否有验证码需要手动处理

### 设备列表加载失败
1. 确保已登录成功
2. 检查网络连接
3. 尝试刷新页面
