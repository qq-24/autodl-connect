# 产品规则

## 产品是什么

AutoDL 一键连接 — Windows 桌面工具，将「登录 AutoDL → 开机 → 复制 SSH → 建隧道 → 打开端口」压缩为一键操作。面向使用 AutoDL 云 GPU 的深度学习/AI 开发者。

## 核心工作流（按顺序）

1. 启动 → 自动加载已保存的设备配置
2. 登录 AutoDL（Selenium 自动化，cookie 持久化免重复登录）
3. 刷新设备列表 → 显示状态、GPU 可用性、备注
4. 一键开机 → 轮询等待设备就绪
5. 建立 SSH 隧道 → 端口转发到 localhost
6. 快捷打开 JupyterLab / AutoPanel

修改任何步骤时，必须确保上下游步骤不受影响。

## 硬性约束

- 仅 Windows，不考虑跨平台
- 单实例、单账号，不支持多账号并行
- 所有操作结果必须在 UI 状态栏即时反馈，禁止静默失败
- 网络操作必须有超时，禁止无限阻塞 UI 线程
- 设备列表刷新不得阻塞用户对其他设备的操作

## UX 规则

以下规则在修改 UI 或用户交互时必须遵守：

- 状态可见：设备状态（开机中/运行中/关机中/已关机）必须实时反映在设备表中，不允许延迟或缺失
- 操作确认：关机、断开连接等破坏性操作必须弹出确认对话框
- 错误信息：必须包含具体原因和建议操作，禁止只显示异常类名或空消息
- 主题一致：深色/浅色切换后所有控件颜色必须同步更新，不留视觉残留
- 布局复用：新增 UI 控件必须复用 `setup_ui()` 中的现有布局模式和交互风格，禁止引入新的布局范式

## 功能边界

修改或新增功能时，必须在以下范围内：

| 功能域 | 允许 | 禁止 |
|--------|------|------|
| 设备管理 | 开机、关机、续费、状态轮询 | 创建/删除实例、镜像管理 |
| 连接 | SSH 隧道、端口转发、断线重连 | VPN、直连公网 IP |
| 自动化 | 登录、设备列表抓取、按钮点击 | 自动训练任务提交、文件传输 |
| 配置 | SSH 参数、凭据、设备映射 | 多账号切换、团队共享配置 |

超出此范围的需求应拒绝或要求用户明确确认。

## 修改决策规则

按优先级排序，修改产品行为时依次考虑：

1. 保持现有用户习惯不变，除非存在明确的可用性缺陷
2. Selenium 选择器修复只改最小范围（AutoDL 网页结构经常变化）
3. 凭据或敏感信息相关变更必须符合安全约束（见 tech.md）
4. 新增功能不得突破上方功能边界表
